    Nginx

    정의

    Nginx(엔진엑스)는 웹 서버 소프트웨어로, HTTP 서버, 리버스 프록시 서버, 메일 프록시 서버, 그리고 일반 TCP/UDP 프록시 서버로 사용할 수 있습니다. Igor Sysoev에 의해 개발되었으며, 경량성과 높은 성능으로 유명합니다.

    기능

    1. 웹 서버: 정적 콘텐츠를 빠르게 제공하며, 동적 콘텐츠의 경우 리버스 프록시를 통해 처리.
    2. 리버스 프록시: 클라이언트 요청을 여러 백엔드 서버로 분산시켜 부하를 분산하고, 보안 및 성능을 개선.
    3. 로드 밸런서: 라운드 로빈, IP 해시, 최소 연결 등 다양한 로드 밸런싱 알고리즘을 지원.
    4. 메일 프록시: IMAP, POP3, SMTP 프로토콜을 지원하여 메일 서버로의 프록시 기능 제공.
    5. 캐싱: 프록시 캐시를 통해 서버 성능을 향상시키고, 자주 요청되는 콘텐츠의 로드 시간을 단축.
    6. SSL/TLS: 보안 연결을 위해 SSL/TLS 암호화를 지원.

    특징

    1. 높은 성능: 비동기 이벤트 기반 아키텍처로 인해 높은 동시성을 제공하고, 적은 자원으로 많은 연결을 처리.
    2. 확장성: 다양한 모듈을 통해 기능을 확장할 수 있으며, 필요에 따라 쉽게 맞춤형 설정 가능.
    3. 유연한 구성: 설정 파일을 통해 매우 유연한 구성 및 관리 가능, 리버스 프록시, 로드 밸런서, 웹 서버 등의 역할을 쉽게 전환 가능.
    4. 안정성: 메모리 누수와 같은 문제가 거의 없으며, 오랫동안 안정적으로 동작.
    5. 다중 플랫폼 지원: Unix 계열 OS 뿐만 아니라 Windows에서도 동작.
